零基础路线

第一次接触 Codex，就按这条路线走

目标不是把所有概念一次讲完，而是让用户先完成一次可验证的安装： 能打开 Codex、能登录、能在一个测试文件夹里让它回答或执行简单任务。

推荐

我只想先用起来

优先用 Codex App 或 IDE 扩展。界面直观，适合第一次体验和日常改代码。

看安装步骤
命令行

我会用 PowerShell

安装 CLI，适合本地项目、脚本、批量操作和后续自动化。

看 CLI 路线
进阶

我的项目在 Linux/WSL

使用 WSL2，把仓库放在 Linux home 目录，减少路径和权限问题。

看 Windows 说明
1

确认你能访问官方服务

先在浏览器打开 ChatGPT 或 OpenAI Platform。国内环境下，如果浏览器都打不开，CLI 登录也大概率会失败。

2

安装 Codex CLI

Windows 打开 PowerShell，复制下面命令。安装完成后重新打开一个终端。

打开安装脚本 
$env:CODEX_NON_INTERACTIVE=1; irm https://chatgpt.com/codex/install.ps1 | iex
3

检查是否安装成功

能看到版本号，就说明命令已经可用。

codex --version
4

登录账号

第一次用建议 ChatGPT 登录。浏览器回调失败时，再用设备码登录。

codex login
codex login --device-auth
5

做一次最小测试

新建一个空文件夹，在里面启动 Codex，让它解释当前目录或帮你创建一个测试文件。先在无关紧要的文件夹里练手。

mkdir codex-test; cd codex-test; codex
