Codex 配置

安装之后，再做这些稳定配置

前面的新手路线负责“跑起来”。下面这些内容负责“长期稳定使用”： 登录方式、国内网络、配置文件、Windows sandbox 和排错。

01

安装 Codex

完全新手优先选 App 或 IDE 扩展；想用命令行时再装 CLI。Windows PowerShell 可使用官方安装脚本：

打开安装脚本 
$env:CODEX_NON_INTERACTIVE=1; irm https://chatgpt.com/codex/install.ps1 | iex

如果你使用 WSL2，则进入 Linux shell 后执行官方 shell 安装脚本。

02

选择登录方式

ChatGPT 登录

适合日常本地开发、桌面 App、IDE 扩展和需要 ChatGPT 工作区权限的场景。

codex login

API Key 登录

适合自动化、CI/CD 或按 API 用量计费的本地流程。密钥只从官方 Platform 创建。

codex login --api-key

无浏览器或 localhost 回调失败时，优先使用设备码登录。

codex login --device-auth
03

国内网络环境建议

Codex 登录、模型调用、网页检索、依赖安装都可能访问海外服务。建议使用稳定、合规、可审计的网络环境， 并确保终端、浏览器、IDE 处在同一可用网络路径下。

  • 优先确认浏览器能访问官方 ChatGPT 与 OpenAI Platform。
  • 公司网络如有 TLS 代理，配置 `CODEX_CA_CERTIFICATE` 指向 PEM 证书包。
  • 不要把 API Key、`~/.codex/auth.json`、访问令牌发给别人或提交到仓库。
  • 网页搜索建议先用 cached 模式，需要最新内容时再改 live。
04

配置 `~/.codex/config.toml`

个人默认配置放在 `~/.codex/config.toml`。项目级配置可放在仓库里的 `.codex/config.toml`， 但只有信任项目后才会加载。 

model = "gpt-5.5"
approval_policy = "on-request"
sandbox_mode = "workspace-write"
web_search = "cached"
model_reasoning_effort = "high"
personality = "friendly"

[windows]
sandbox = "elevated"

[shell_environment_policy]
include_only = ["PATH", "HOME", "USERPROFILE", "TEMP", "TMP"]
05

Windows 推荐设置

Windows 11 是推荐环境。原生 Windows 优先使用 `elevated` sandbox；如果企业设备策略限制，再考虑 `unelevated` 或 WSL2。

原生 elevated推荐，性能好，隔离更强
原生 unelevated无管理员权限时的备选
WSL2适合 Linux 工具链和仓库已在 WSL 的团队
06

排错清单

使用 `codex login --device-auth`，或检查 localhost 回调是否被安全软件、代理、远程环境拦截。

如公司网络有 TLS 检查，把企业根证书 PEM 路径配置到 `CODEX_CA_CERTIFICATE`。

先验证系统代理、包管理器源和 DNS。需要联网执行任务时，只允许必要域名和方法。

官方依据

本页配置项参考 OpenAI Codex 官方手册中的认证、配置、环境变量、Windows 与网络访问章节。 继续维护时，建议定期核对官方文档。
